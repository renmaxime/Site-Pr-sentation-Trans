C'est quoi la sonification ?

Transformer des données en son

La sonification, c'est l'art de représenter des données par le son. Comme une visualisation, mais audible — utile en science, en accessibilité, en art.

Donnéestableau, image, texte, nombre…

Son généréfréquence, rythme, timbre…

Sonifcell est un logiciel développé par Maxime poret qui permet de créer ces programmes sonores visuellement, en connectant des blocs-fonctions entre eux. Chaque bloc réalise une opération — charger un fichier, transformer un nombre, générer une onde sonore.

Semestre 1

Refonte de l'arborescence

Repenser la classification des fonctions de Sonifcell selon une logique cognitive, et la valider par des tests utilisateurs.

L'arborescence originale de Sonifcell ne reflétait pas la façon dont les utilisateurs pensent naturellement. L'objectif du semestre 1 : repartir de zéro pour proposer une classification plus intuitive, puis la tester pour mesurer son efficacité.

Les 5 étapes

1

Prise en main de Sonifcell

Exploration complète du logiciel — comprendre chaque fonction, ses paramètres et ses cas d'usage.

2

Deux arborescences "from scratch"

Création indépendante de deux nouvelles classifications sans influence de l'arborescence originale.

3

Tri de cartes — SCRIME

Méthode UX de card sorting réalisée avec des participants pour valider la pertinence des regroupements.

4

Outil de test utilisateur

Développement d'une application web mesurant le temps et le nombre de clics pour trouver une fonction dans chaque arborescence.

5

Analyse et sélection

Comparaison des performances des arborescences — la plus efficace a été retenue pour le wiki du semestre 2.

Ce qu'on a appris

🧠

Ergonomie cognitive

Comment les utilisateurs catégorisent naturellement des objets, et comment concevoir une classification qui suit cette logique.

🃏

Méthode card sorting

Technique UX de tri de cartes permettant de recueillir des données sur la façon dont les utilisateurs organisent l'information.

📊

Tests utilisateurs

Conception et analyse de tests mesurant objectivement la performance d'une interface.

💻

Développement web

Création de l'outil de test utilisateur en ligne — première expérience de développement web du projet.

Semestre 2

WikiSonifcell

Construction d'un wiki de référence complet pour Sonifcell — entièrement en HTML, CSS et JavaScript.

L'objectif : qu'un utilisateur sans formation puisse comprendre Sonifcell, trouver n'importe quelle fonction et réaliser ses propres programmes sonores, uniquement en consultant le wiki.

Fonctionnalités du wiki

📖

Référence des fonctions

75+ fonctions documentées — description, inputs typés, outputs, cas d'usage.

⚡

Exemples concrets

5 cas d'usage pas à pas — Morse, souris via UDP, gamme, écart à la référence.

🗺️

Carte des zones mécaniques

Carte interactive des 5 zones de l'interface Sonifcell avec explications au clic.

📋

Guide d'utilisation

Astuces pratiques pour prendre en main l'interface — connexions, réglages, raccourcis.

🔗

Navigation cohérente

Fil d'Ariane, liens entre pages, arborescence dépliable, paramètres d'URL dynamiques.

Comment ça marche

definition.js — page dynamique
// URL : definition.html?fonction=Atan let params = new URLSearchParams( window.location.search ); let nom = params.get("fonction"); // Un objet pour 75+ fonctions const fonctions = { "Atan": { description: "Arc tangente...", input: ["(float) x0"], output: "(float) — radians" }, // 74 autres... }; // Affichage dans la page let titre = document .createElement("h1"); titre.textContent = nom; document.querySelector(".content") .appendChild(titre);

Une seule page HTML pour 75 fonctions — le contenu est généré dynamiquement selon le paramètre dans l'URL.

Ce qu'on a appris

🛠️

HTML / CSS / JavaScript

Manipulation du DOM, génération dynamique de contenu, architecture CSS, paramètres d'URL.

🔀

Git & collaboration

Branches par fonctionnalité, pull requests, résolution de conflits, base de code partagée.

📚

Documentation technique

Comprendre et reformuler des fonctions techniques pour des non-experts — vulgarisation et rigueur.

DDRS

Développement Durable & Responsabilité Sociétale

Comment ce projet s'inscrit dans une démarche durable, inclusive et responsable.

🌱

Écologique

Sur le plan environnemental, le choix d'un site statique (sans base de données ni serveur actif dédié) permet de limiter naturellement l'empreinte carbone liée à l'hébergement et à la consultation

♿

Inclusion

Le wiki est rédigé en anglais pour toucher un public international. Sonifcell lui-même est un outil d'accessibilité — il permet de rendre des données perceptibles par le son, ce qui est particulièrement utile pour les personnes malvoyantes. Notre wiki étend cette mission en rendant l'outil plus accessible.

🌍

Sociétal

Ce wiki vise à démocratiser l'accès à la sonification, une technologie utile pour la recherche, l'art et la pédagogie. En simplifiant la prise en main de ces concepts, l'outil facilite le partage et offre une nouvelle manière de percevoir et de transmettre des données complexes.

💡

Économique

Le projet repose exclusivement sur des technologies gratuites et open source (VS Code, Git, GitHub, Google Fonts). Cette approche technique garantit la gratuité totale de l'hébergement et du développement, assurant la viabilité et la disponibilité à long terme du site sans dépendance financière.

Bilan

Ce qu'on retient de ce projet

Un projet qui nous a permis de mener une démarche complète — de la recherche utilisateur à la livraison d'un produit fonctionnel.

🛠️

Bilan technique

Ce projet nous a permis de comprendre ce que signifie vraiment structurer un projet web : organiser des fichiers, gérer des dépendances entre pages. Nous avons approfondi JavaScript au-delà des bases. Mais au-delà du code, il fallait d'abord comprendre Sonifcell en profondeur : tester les fonctions, comprendre leurs interactions, et les faire fonctionner ensemble pour construire des exemples réalistes. La partie technique du développement ne représentait qu'une partie du travail.

🤝

Bilan personnel

Travailler pour un vrai client, collaborer en binôme sur plusieurs mois, gérer nos priorités sous contrainte de temps — ce projet nous a confrontés aux vraies conditions d'un projet professionnel. La répartition des tâches a bien fonctionné dès le début, ce qui nous a permis d'avancer en parallèle sans friction. Et surtout, créer quelque chose d'utile et d'utilisable a rendu le travail vraiment motivant.

🔄

Ce qu'on referait

La répartition des tâches a bien fonctionné dès le départ et c'est clairement quelque chose qu'on referait. On referait aussi cette phase de recherche approfondie sur Sonifcell — comprendre chaque fonction, la tester, comprendre comment les faire interagir — parce que c'est elle qui donne du sens à tout le reste.
